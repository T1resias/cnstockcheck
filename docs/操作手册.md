# CN-STOCK A股每日复盘 — 操作手册

## 一、网站功能

| 标签页 | 内容 | 数据来源 |
|--------|------|----------|
| **涨停股票** | 当日涨停个股，按行业题材归类，可筛选 | 新浪财经 + 东方财富 |
| **题材复盘** | 概念板块涨幅排名，连续上榜板块高亮标记 | 东方财富 / 本地聚合 |
| **财经资讯** | 当日重点财经新闻 | 新浪财经 |

### 涨停股票页
- 顶部**题材标签**可点击筛选，比如只看"半导体"、"汽车零部件"
- 表格列：代码、名称、最新价、涨幅、连板数、首次封板时间、换手率、封单资金、题材
- 连板 ≥3 天：红色标记；连板 1-2 天：黄色标记

### 题材复盘页
- 概念板块按涨幅降序排列，展示前 30 名
- **连续 3 天以上**进入涨幅前 20 的板块，顶部有醒目高亮卡片
- 排名行中 🔥 连榜标记（🔥🔥🔥 = 4天以上，🔥🔥 = 3天，🔥 = 2天）

### 日期切换
- 页面顶部下拉框可选择历史日期
- 或直接访问 `?date=YYYY-MM-DD`（如 `?date=2026-05-06`）

## 二、刷新数据

### 网页端刷新
点击页面右上角 **"刷新数据"** 按钮，等待提示 "刷新成功"。

> **注意**: 刷新会调用外部 API 拉取最新行情，需要 5-45 秒（视乎东方财富接口是否限流）。刷新成功后页面会自动更新。

### 本地命令行刷新
```bash
npm run refresh
```

### 线上自动刷新
GitHub Actions 每个**交易日北京时间 15:30** 自动运行刷新，并将新数据提交到仓库，Vercel 自动部署。

## 三、数据来源

### 1. 涨停股票
```
新浪财经 Market_Center.getHQNodeData
URL: vip.stock.finance.sina.com.cn/quotes_service/api/json_v2.php/Market_Center.getHQNodeData
参数: node=hs_a (全部A股), sort=changepercent (按涨幅排序)
策略: 分页拉取（每页80只），直到涨幅降到 9.5% 以下停止翻页
筛选: 主板 ≥9.9%，创业板/科创板 ≥19.9%，北交所 ≥29.9%
过滤: 自动排除 ST 股票
```

### 2. 题材板块排行
```
主数据源: 东方财富 push2.eastmoney.com/api/qt/clist/get
参数: fs=m:90+t:3 (概念板块), fid=f3 (按涨幅排序)
备选方案: 如果东方财富不可用，从涨停股票的行业分布自动聚合板块排名
```

### 3. 财经资讯
```
新浪财经 feed.mix.sina.com.cn/api/roll/get
参数: lid=2509 (财经新闻栏目), num=20
```

### 4. 个股行业分类
```
东方财富 push2.eastmoney.com/api/qt/stock/get
参数: secid=<市场>.<代码>, fields=f57,f127
f127 = 申万行业分类（如"半导体"、"汽车零部件"）
缓存策略: 本地 data/industry-map.json，30天自动过期重查
```

### 5. 交易日历
```
内置: 2025-2026 年中国A股节假日列表
在线更新: github.com/rainx/cn_stock_holidays (自动拉取最新)
判断逻辑: 周一至周五 且 非节假日 = 交易日
```

## 四、架构逻辑

### 数据流
```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 新浪财经 API  │    │ 东方财富 API  │    │ 新浪资讯 API  │
│  (涨停股票)   │    │ (板块排行)    │    │  (财经新闻)   │
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       │                   │                   │
       └───────────────────┼───────────────────┘
                           │
                    ┌──────▼──────┐
                    │  refresh.ts │  ← 编排器
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼──┐  ┌─────▼─────┐  ┌──▼────────┐
       │ 连榜计算 │  │ 行业映射  │  │ 数据存储   │
       │streak   │  │industry   │  │data-store │
       └──────┬──┘  │   map     │  └─────┬──────┘
              │     └───────────┘        │
              │                   ┌──────▼──────┐
              │                   │  本地: JSON  │
              └───────────────────┤  Vercel:Blob │
                                  └─────────────┘
```

### 刷新流程 (refresh.ts)
1. 判断当前北京时间是否**交易日 + 15:00 后**
2. 如果是 → 拉取今日数据；否则 → 拉取最近交易日数据
3. 并行调用：涨停池 + 板块排行 + 资讯
4. 如果东方财富板块 API 不可用 → 从涨停股行业分布聚合板块排名
5. 计算连榜数据（与昨日对比，同一板块出现则 streak+1）
6. 保存到 JSON（本地）或 Vercel Blob（线上）

### 连榜计算逻辑 (consecutive-days.ts)
- 每日刷新时，取**当日涨幅前 20 的板块**
- 与之前的连榜记录对比：
  - 同一板块今日也在前 20 → 连榜天数 +1
  - 不在前 20 → 连榜中断，归零
- 前端展示：≥2 天火焰标记，≥3 天顶部高亮

### 本地 vs 线上存储 (data-store.ts)
```
本地开发: data/history/YYYY-MM-DD.json  (文件系统)
线上Vercel: Vercel Blob Storage  (有 BLOB_READ_WRITE_TOKEN 时)
           → 读: list() + fetch url
           → 写: put(path, content)
Vercel 降级: 编译时嵌入的 JSON (仅静态数据，不更新)
```

## 六、常见问题

### 刷新失败提示"网络错误"
- Vercel 免费版有 10 秒超时限制 → 行业查询自动跳过，仅用缓存
- 东方财富 API 可能限流 → 等待 30-60 分钟自动恢复

### 涨停数量比别的网站少
- 可能原因：API 分页过程中东方财富中断 → 重新刷新
- ST 股票已自动过滤
- 北交所股票需要 ≥29.9% 才计入涨停

### 股票行业显示"其他"
- 该股票不在本地行业缓存中，且东方财富 API 当时不可用
- 行业缓存随每日刷新自动增长，下次刷新时东方财富恢复即可补充

### 日期显示不正确
- Vercel 服务器在美国，但系统已处理为北京时间
- 如果仍不对，可能是 ISR 缓存（最多 6 小时），等待缓存过期或手动加 `?nocache=1`
